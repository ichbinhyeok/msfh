# Route Promotion Board

## Purpose
This board exists so future agents do not rely on founder memory.

When metrics exist, the agent should review this board first and then tell the user whether any route deserves promotion.

## Current status
No metrics yet.

Default state for all held route families is `hold`.

## Held route families

### No-recommended-improvements
- Current status: `hold`
- Current surface: support or `noindex`
- Promote when:
  - indexed report or choose-project routes show repeated no-recommendations confusion
  - the page can route users to insurance-discount, local alternative, or trust content without becoming a dead page
- Do not promote when:
  - it only captures zero-action curiosity

### Group-5 and prioritization problem states
- Current status: `hold`
- Current surface: support or `noindex`
- Promote when:
  - user messages, contact notes, or search evidence show repeated demand
  - the page can help without becoming a pure status mirror
- Do not promote when:
  - it duplicates the Support Center with no added decision value

### RFI and portal-status routes
- Current status: `hold`
- Current surface: support or `noindex`
- Promote when:
  - admin notes show repeated RFI confusion from real users
  - the page can give a corrective checklist rather than just status definitions

### Final inspection and draw-request routes
- Current status: `hold`
- Current surface: support or `noindex`
- Promote when:
  - the site already has users moving deep into contractor and project completion stages
  - the route can support a real operational bottleneck

### County overlays
- Current status: `hold`
- Current surface: not built or `noindex`
- Promote when:
  - statewide routes prove demand
  - county-specific contractor density or threshold differences justify the page
  - maintenance burden is justified by commercial density

### Partner finder pages
- Current status: `hold`
- Current surface: not built or `noindex`
- Promote when:
  - improvement-specific routes already have demand
  - partner coverage exists
  - the page adds routing value rather than becoming a thin directory
