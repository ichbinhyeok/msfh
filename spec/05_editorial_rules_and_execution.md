# 05 Editorial Rules And Execution

## Editorial promise
The site explains the likely next action after an MSFH inspection report or recommendation. It does not pretend to be the program, and it does not pretend to guarantee approval.

## Voice rules
- calm
- specific
- program-aware
- scenario-first
- never scammy

## Mandatory language rules
Use:

- `may`
- `likely`
- `verify`
- `based on your report`
- `under the current program rules`
- `the program may deny`

Avoid:

- `guaranteed approval`
- `free roof`
- `approved contractor` unless the current source explicitly supports that wording
- `the program will definitely pay`
- `every window` or `every roof` language without source support

## Claims the site can make
- inspection and grant are separate
- only recommended and later observed improvements are eligible
- the homeowner is responsible for choosing and managing the contractor
- some report recommendations require careful reading before quote shopping
- roof replacement can be tied to eligible work, but not every roof project is automatically covered
- attached or townhouse homes may face narrower eligible scope under the program

## Claims the site must not make
- grant approval certainty
- reimbursement certainty
- official program affiliation
- exact premium savings promises for a specific home
- exact cost promises without clear assumptions

## Page-family rules

### Inspection-report page
- must explain what the report contains
- must tell the user what to read first
- must route into improvement or project-choice pages quickly

### Choose-project page
- must compare likely first projects honestly
- must keep program-alignment visible
- must explain why the best project is not always the most expensive project
- must surface attached-home or townhouse caveats when they change the answer

### Contractor-quote page
- must explain contractor-type fit
- must tell the user to verify license and scope
- must avoid pressuring users into signing anything early

### Improvement page
- must explain what the recommendation means
- must explain what commonly causes denial or confusion
- must show what the program may cover and what it will not cover
- must separate detached-home roof scenarios from attached-home opening-protection-only scenarios when current rules require it

### Support page
- must give a corrective next step
- must link back into the main decision wedge
- must not become a pure FAQ clone

## Execution order
1. Home and trust pages
2. Program decision pages
3. Improvement pages
4. Evergreen guides
5. Held support routes

## Definition of done for a public page
- has a clear trigger state
- has a program-aware answer
- has a source stack
- has one primary CTA
- has a `what not to assume` section
- has at least one internal link to the next likely route

## Editorial kill rule
If a draft page reads like a generic Florida home-improvement article or a fake official page, stop and rewrite the angle.
