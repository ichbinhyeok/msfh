# 00 Strategy

## Positioning
Build a Florida-only post-inspection decision site for homeowners moving through the My Safe Florida Home workflow after the initial inspection.

This is not:

- a generic hurricane-hardening site
- a broad Florida insurance blog
- an official-program FAQ mirror
- a contractor directory pretending to be editorial

This is:

- a post-inspection decision engine
- a project-priority and quote-prep workflow product
- an improvement-specific lead-routing layer

## Core wedge
The full category includes:

- inspection report understanding
- project priority
- what the program pays for
- contractor choice and quote comparison
- support states like no recommendations, group 5, and RFI

## Phase 1 wedge
Phase 1 should be narrower than the full category.

The first wedge is:

- report received
- recommendation exists or is being interpreted
- project choice and contractor choice now matter

If the first release does not feel like a `post-inspection decision engine`, it is still too broad.

## Why this can work
- The state program already creates demand and urgency.
- Official pages explain process but do not fully solve contractor-side decision work.
- The program explicitly leaves contractor choice to the homeowner.
- The improvement categories create strong homeowner decision demand:
  - opening protection
  - roof-to-wall
  - roof deck
  - SWR or roof-related work

## Why now
- Current program pages still show active inspections and grants.
- Recent official updates show large program scale and continuing policy interest.
- Search results already show contractor marketers crowding the category, which proves strong post-report homeowner demand.
- A decision-layer product can sit between official process content and contractor sales pages.

## The strategic bet
The bet is not that `MSFH` branded head terms are easy.

The bet is that this is a **high-intent branded-plus-decision market** where:

- the report creates real uncertainty
- the homeowner has to spend real money soon
- trust and clarity can win even without giant traffic

## The market shape
This is a `single-state program market`.

That is a feature, not a bug.

- it narrows implementation
- it makes sources easier to audit
- it keeps the content system honest
- it allows county overlays later only when they are deserved

## What must not happen
The product fails if it drifts into any of these:

1. broad pre-inspection eligibility SEO
2. exact FAQ duplication of the Support Center
3. generic impact-window or roofing pages with no program decision angle
4. scammy `free roof` framing
5. county pages with no real local reason

## Strategic conclusion
The strongest product is not `My Safe Florida Home Eligibility Checker`.

It is:

`Florida post-inspection decision engine for recommended improvements, project choice, and contractor quote routing.`

## Product layering
The main product should stay homeowner-first.

- Core layer: report understanding, project choice, and homeowner quote prep
- First workflow asset: homeowner-built opening-protection quote-prep brief
- Optional layer: reuse of the same brief across homeowner email or text outreach when the same first-send explanation is needed again

## Interaction rule
The product should feel like a next-step assistant, not an organized information shelf.

- The first screen should answer `what do I do now?`
- The second answer should be `what do I need ready?`
- The third answer should be `what should I not do yet?`
- When the answer is `start quote prep`, the product should carry the known context forward so the homeowner does not have to restate the same report, home-type, or scope cues
- Longer explanation, source proof, and route comparison belong underneath the immediate action guidance
- If a page leads with taxonomy, summaries, or route jargon instead of a clear next move, the page is still too editorial

This means the product should not behave like a broad contractor suite.

- The homeowner should be able to build and carry the brief without extra setup.
- Any behind-the-scenes follow-up is secondary and should never define the public product.
- Internal-only notes, reusable checklists, and follow-up aids stay behind the free homeowner brief.

## 90-day objective
Within 90 days, the site should prove three things:

1. it can index and earn impressions for report and improvement routes
2. users click quote-checklist and contractor-type CTAs
3. at least a few qualified improvement-specific leads arrive without phone-heavy operations

## Success condition
Success is not page count.

Success is:

- a narrow report-to-project wedge
- clear improvement-specific routing
- clear homeowner decision intent by improvement type

## Failure condition
Failure is:

- broad branded traffic with no decision intent
- thin FAQ clones
- generic Florida home-improvement pages
- too many support routes before the main wedge proves traction
